# Ticketing Model

This project uses a lightweight ticketing model that feels realistic without requiring Jira or ServiceNow.

## Recommended tooling

- GitHub Issues for ticket intake
- GitHub Projects for workflow tracking
- Pull Requests for approval and implementation
- Markdown evidence files for audit trail

## Ticket states

- `new`
- `triage`
- `approved`
- `in-progress`
- `verification`
- `done`
- `rolled-back`

## Ticket fields

- type
- requester
- environment
- business reason
- affected resources
- risk
- rollback plan
- approval
- execution notes
- evidence
