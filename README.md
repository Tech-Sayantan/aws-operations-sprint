# AWS Operations Sprint

This workspace is a 7-day AWS CloudOps practice setup built around real ticket-driven operations.

## Goal

Practice the kind of work a CloudOps team handles in a real company:

- access requests
- secure S3 bucket setup
- security group changes
- EC2 and EBS operations
- package install and config changes
- monitoring and audit evidence

## Suggested workflow

1. Create a ticket in `tickets/`
2. Classify the request
3. Design the change
4. Implement it with Terraform or Ansible
5. Validate it
6. Attach evidence
7. Close the ticket

## Folder map

- `tickets/` - request templates and sample tickets
- `syllabus/` - 7-day learning plan
- `docs/` - conceptual notes and runbooks
- `terraform/` - infrastructure as code
- `ansible/` - configuration and server ops automation
- `evidence/` - change logs, screenshots, and verification notes
